# 首页移动端适配设计说明

## 目标

- 修复首页在手机竖屏下显示不全、底部导航遮挡内容、模块过于拥挤的问题。
- 保持桌面端与大屏平板现有蓝白工作台风格不变。
- 优先保证手机端首页信息完整显示、滚动顺畅、首屏层级清晰。

## 问题定位

- `Layout.tsx` 的主内容区与底部导航同时占用可视高度，导致首页底部内容容易被遮挡。
- `Dashboard.tsx` 头部概览区在小屏上纵向占用过大，右侧摘要卡片在窄屏下过于紧凑。
- 统计卡片默认两列且卡片内边距较大，在小宽度设备上容易显得拥挤。
- 首页下半部分使用 `flex-1 min-h-0` 的大屏布局思路，在移动端会放大“区域抢高度”的问题。

## 方案对比

### 方案 A：单列优先

- 手机端将概览摘要、统计卡片、列表区域按纵向信息流重新组织。
- 普通手机宽度允许统计卡片使用舒适两列，超窄屏再降为单列。
- 优点是风险最小，最容易保证显示完整与阅读舒适。

### 方案 B：保留双列压缩

- 保留现有双列结构，只缩小字号、边距和卡片高度。
- 改动较小，但在更窄的手机屏幕上仍可能显得拥挤。

### 方案 C：移动端完全重排

- 为移动端定制首页模块顺序与独立视觉结构。
- 体验最好，但实现范围更大，当前需求没有必要。

## 选定方案

- 采用方案 A。
- 手机端优先保证可读性与完整性，桌面端只做最小影响的响应式增强。

## 组件调整

### Layout

- 给主内容区提供更明确的底部留白，避免固定底部导航遮挡首页最后一屏。
- 缩小移动端底部导航的横向占用与文字密度，减少对可视区域的压迫感。
- 调整页面内容容器在移动端的内边距与底部安全区策略。

### Dashboard 顶部概览

- 降低 Hero 区纵向高度与内边距。
- 将右侧两个摘要卡在手机端改为单列堆叠，在平板以上恢复双列。
- 限制标题与描述的换行密度，避免首屏被文字块撑得过高。

### Dashboard 统计卡

- 在极窄屏改为单列，在常见手机宽度使用两列。
- 压缩卡片内边距和数字字号，保持信息完整但不拥挤。

### Dashboard 下半区

- 移动端取消“左右分栏 + 抢占剩余高度”的布局，改成自然纵向流式排列。
- 仅在大屏保留 `lg:grid-cols-3` 和主次分栏。

## 验证标准

- 首页在常见手机竖屏宽度下无横向溢出。
- 底部导航不遮挡首页最后一个模块。
- 首屏可完整看见品牌头部、首页概览和主要摘要信息。
- 桌面端现有布局风格保持稳定，不引入新的视觉回退。
