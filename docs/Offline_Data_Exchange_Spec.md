# 线下场景数据交换接口规范 (Offline Data Exchange Specification)

本系统设计为支持纯线下运行环境。为了在无网络、无中心数据库的情况下，实现跨工位、跨设备的数据流转与存档，系统通过标准化命名的 Excel (`.xlsx`) 文件进行数据的导入与导出。

## 1. 导出/导入文件命名规范

文件命名规则需包含与系统界面完全同步的**业务标识**与**唯一流水号 (ID)**，以便线下归档和识别。

### 1.1 工艺文件导出
- **格式**: `工艺文件_[工艺ID]_[工艺名称].xlsx`
- **示例**: `工艺文件_proc-1_铝合金箱体加工及焊接工艺.xlsx`
- **包含内容**: 
  - 包含工艺流程所有节点（步骤）。
  - **字段**: `序号`, `工序名称`, `工序内容`, `参数要求`, `操作要求`, `需要拍照`, `关键工序`, `设备`, `工具`, `耗材`。

### 1.2 生产执行跟踪卡导出
- **格式**: `执行跟踪卡_[批次号]_[工单ID].xlsx`
- **示例**: `执行跟踪卡_B-260424-A_WO-20260424-001.xlsx`
- **包含内容**: 
  - 该批次当前所有的执行状态与填报数据。
  - **字段**: `工序序号`, `工序名称`, `工序内容`, `参数要求`, `操作要求`, `状态` (未完成/待检验/已检验), `操作人`, `操作记录`, `现场照片数`, `检验人`。

## 2. 离线图片与签名数据的处理

由于 Excel (.xlsx) 本身并不适合作为大型图片的存储介质（可能导致文件损坏或难以导入解析），并且系统处于离线状态，所以我们对图片与签名的处理接口如下：

### 2.1 现场照片存储
- **当前机制**: 在浏览器环境内以 Base64 格式持久化存储（通过 IndexedDB）。
- **导出展现**: 导出的 Excel 中仅记录**照片数量**。
- **扩展建议**: 若后续需要将照片提取到操作系统本地，可在“生产执行详情”中增设一个 `ZIP` 打包导出功能，将所有 Base64 图片还原为 `.jpg` 格式，命名规则为：`照片_[工单ID]_[工序序号]_[时间戳].jpg`。

### 2.2 离线电子签名与检测章
- **流转方式**: 签名与印章通过个人中心直接绘制或上传图片生成。
- **存储与识别**: 管理员可通过统一的系统分配给各个终端，或者在同一台物理终端（如车间共用平板）上，通过账号体系直接隔离调用，无需依赖外部网络传输。

## 3. Excel 数据导入接口规范

导入功能基于 `xlsx` 库对表格进行按行解析，系统对导入数据的要求如下：

### 3.1 导入新工艺文件
系统会读取 Excel 表格的**第一个工作表 (Sheet1)**。
必须包含以下列标题（表头）：
*   `工序名称` (必填)
*   `工序内容`
*   `参数要求`
*   `操作要求`
*   `需要拍照` (填入 "是" 或 "否")
*   `关键工序` (填入 "是" 或 "否")

### 3.2 导入跟踪卡 (初始化新任务)
系统读取 Excel 的文件名称作为**产品名称**，并自动生成批次号与初始化未开始的执行卡片。关联的工艺文件需预先在“工艺管理”中存在（默认选取第一个匹配的工艺模板进行装配）。
